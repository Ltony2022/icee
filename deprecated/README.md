This folder contains outdated code and resources for historical reference and backward compatibility.

## Purpose 🎯

-   **Reference:** Understand the project’s evolution.
-   **Legacy Support:** Aid in debugging and migration.
-   **Documentation:** Provide context for old implementations.

## Structure 🗂️

-   **Modules:** Old source code files.
-   **Docs:** Legacy documentation and changelogs.
-   **Resources:** Config files, scripts, etc.

## Usage ⚠️

-   **Read-Only:** Avoid modifying deprecated files.
-   **Reference Only:** Use for historical context or troubleshooting.
-   **Document References:** Comment on any deprecated code used.

## Deprecation Process 🛠️

1. **Identify:** Find obsolete code.
2. **Move:** Place it in the `deprecated` folder.
3. **Open a PR:** Document the changes.
4. **Update:** Replace references to the new code.
   We don't delete deprecated code to maintain historical context (except for some reasons like security issues, etc).
